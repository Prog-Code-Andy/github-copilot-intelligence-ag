# Contributing

Thank you for your interest in contributing to this repository.

## How to Contribute

- Open an issue to discuss major changes before implementing them.
- Fork the repository and create a feature branch for your changes.
- Ensure your branch name is descriptive and contextual.
- Submit a pull request with a clear title and description.

## Pull Request Guidelines

- Describe the purpose of the change.
- Link any related issues.
- Include testing or verification steps.
- Keep changes focused and avoid unrelated modifications.

## Code Style

- Follow the existing conventions in the repository.
- Be consistent with formatting and naming.
- Add documentation for any new features or behavior.

## Communication

- If you are unsure about a change, ask before implementing it.
- Be respectful and constructive in discussions.

## Issue Reporting

Provide a clear description of the problem, steps to reproduce, and expected vs actual behavior.
