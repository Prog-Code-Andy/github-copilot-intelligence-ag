# Security Policy

If you discover a security vulnerability in this repository, please report it immediately.

## Reporting a Vulnerability

1. Create a private issue in this repository if possible.
2. If the repository is public and private issues are unavailable, report through GitHub Security Advisories or email the repository maintainers directly.

## Supported Versions

This repository currently tracks the latest version of its contents. Security issues should be reported for the current main branch.

## What to Include

Please provide:

- A clear description of the issue.
- Steps to reproduce the vulnerability.
- Any relevant proof of concept or exploit code.
- The impact of the vulnerability.

## Response

The maintainers will review reports as soon as possible and respond with next steps.

## Disclosure

Do not disclose security issues publicly until they have been responsibly disclosed and addressed.
